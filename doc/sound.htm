Sound

PyTraffic uses SDL (via the SDL_mixer library) for sound effects and music playback. On modern Linux systems SDL automatically selects the appropriate audio backend — typically PipeWire or PulseAudio — so no manual configuration is needed.

Troubleshooting

If you get no sound, check the following:

  • Make sure your system audio daemon is running. On most distributions this is PipeWire (with the pipewire-pulse compatibility layer) or PulseAudio. You can verify with pactl info.
  • Check that PyTraffic is not muted in your desktop volume mixer (e.g. pavucontrol, pw-top, or the system tray volume applet).
  • Ensure the SDL_mixer library is installed. On Fedora: dnf install SDL2_mixer; on Debian/Ubuntu: apt install libsdl2-mixer-2.0-0.
  • If SDL is picking the wrong audio device you can override it by setting the SDL_AUDIODRIVER environment variable before launching PyTraffic, for example: 
    SDL_AUDIODRIVER=pipewire python3 Main.py
    Common values are pipewire, pulseaudio, and alsa.
